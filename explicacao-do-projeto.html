App de Previsão do Tempo com IA

Explicação completa de todos os arquivos do projeto

1. Visão geral do projeto

Este é um aplicativo web de previsão do tempo. A pessoa digita (ou fala) o nome de uma cidade e o app mostra a temperatura e a umidade daquele lugar. Além disso, há um botão que pede a uma inteligência artificial uma sugestão de qual roupa usar com base no clima.

2. O arquivo index.html — a estrutura

O HTML monta os elementos visíveis da página. Os principais são:

  • Campo de texto (input-cidade) — onde o usuário digita o nome da cidade. O placeholder é a dica em cinza ("Digite a cidade").
  • Botão da lupa 🔍 — ao ser clicado, executa a função cliqueiNoBotao() para buscar o clima.
  • Botão do microfone 🎤 — ao ser clicado, executa detectaVoz() para buscar o clima falando.
  • Caixa de resultado (caixa-media) — começa vazia; é nela que o resultado aparece.

No final do arquivo, a linha <script src="scripts.js"></script> conecta o HTML ao código JavaScript, dando "vida" à página.

3. O arquivo styles.css — a aparência

O CSS define como cada parte da página se parece. Pontos importantes:

  • Seletor * — zera margens e espaçamentos de todos os elementos, para começar de um ponto neutro.
  • body — coloca uma imagem de fundo (praia e montanhas) e usa display: flex para centralizar a caixa principal na tela.
  • .caixa-maior — a caixa preta semitransparente. Usa width: 90% com max-width: 450px, o que a torna responsiva: se adapta ao celular e mantém um tamanho agradável no computador.
  • .botao — botões redondos (border-radius: 50%) com efeito de crescer ao passar o mouse (:hover).
  • .img-botao — define height: 24px e width: auto, garantindo que os ícones da lupa e do microfone fiquem do mesmo tamanho e proporcionais.
  • .botao-ia — o botão azul que pede a sugestão de roupa para a IA.

4. O arquivo scripts.js — o cérebro

É aqui que está toda a lógica. O arquivo começa com um comentário que descreve o algoritmo — o passo a passo planejado antes de programar. Existem três funções principais.

4.1 — função cliqueiNoBotao()

Executada ao clicar na lupa. O passo a passo dela é:

  1. Pega o texto digitado no campo da cidade.
  2. Monta o endereço (URL) do servidor de clima.
  3. Usa fetch() para buscar os dados no servidor.
  4. Converte a resposta para o formato JSON (um formato fácil de ler no JavaScript).
  5. Escreve o resultado dentro da caixa usando innerHTML.

Por que async e await?

Buscar dados num servidor demora. A palavra await ("espere") faz o código pausar até a resposta chegar, evitando continuar com dados que ainda não existem. Toda função que usa await precisa ser marcada como async.

4.2 — função detectaVoz()

Executada ao clicar no microfone. Ela:

  1. Liga o reconhecimento de voz do navegador, no idioma português (pt-BR).
  2. Ouve o que a pessoa fala e transforma em texto (transcrição).
  3. Coloca esse texto no campo da cidade.
  4. Chama automaticamente cliqueiNoBotao() para buscar o clima.

4.3 — função pedirSugestaoRoupa()

Executada ao clicar no botão azul. Ela:

  1. Lê a temperatura, a umidade e a cidade que já estão na tela.
  2. Envia esses dados para uma IA (Groq) com um pedido de sugestão de roupa.
  3. Recebe a resposta da IA e a exibe na tela.

A linha que mostra a resposta é:

document.querySelector(".resposta-ai").innerHTML = dados.choices[0].message.content

Lê-se assim: "encontre o elemento .resposta-ai na página e coloque dentro dele o texto da resposta da IA". O caminho dados.choices[0].message.content navega dentro do objeto JSON até chegar ao texto final.

⚠️ Dica de segurança

Chaves de API não devem ficar visíveis no código que vai para a internet, pois qualquer pessoa poderia copiá-las e usá-las. Em projetos de estudo isso é aceitável, mas em um app real elas devem ficar protegidas em um servidor.
